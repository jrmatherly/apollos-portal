The Apollos AI backend exposes a REST API at `/api/v1/` for managing authentication, provisioning, API keys, teams, usage, models, and settings.

## Base URL

```
http://localhost:8000/api/v1
```

In production, replace with your deployed backend URL.

## Authentication

All endpoints (except `/health` and `/ready`) require a valid Microsoft Entra ID access token in the `Authorization` header:

```bash
curl -H "Authorization: Bearer <access-token>" \
  https://your-backend/api/v1/me
```

## Endpoints

### User endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/api/v1/me` | GET | Get the current user's profile (OID, email, name, roles, admin status) |
| `/api/v1/status` | GET | Check provisioning status |
| `/api/v1/provision` | POST | Provision the current user (map Entra ID groups to LiteLLM teams) |
| `/api/v1/keys` | GET | List the current user's API keys |
| `/api/v1/keys/new` | POST | Create a new API key |
| `/api/v1/keys/{key_id}/rotate` | POST | Rotate an API key |
| `/api/v1/keys/{key_id}/revoke` | POST | Revoke an API key |
| `/api/v1/teams` | GET | List the current user's teams |
| `/api/v1/usage` | GET | Get usage statistics |
| `/api/v1/models` | GET | List available models for the user's teams |
| `/api/v1/settings` | GET | Get user settings |
| `/api/v1/settings` | PATCH | Update user settings |

### Admin endpoints

Admin endpoints require the `portal_admin` role in Entra ID app roles.

| Endpoint | Method | Description |
|---|---|---|
| `/api/v1/admin/users` | GET | List all provisioned users (paginated) |
| `/api/v1/admin/users/{user_id}` | GET | Get detailed user info with keys, teams, and audit history |
| `/api/v1/admin/users/{user_id}/deprovision` | POST | Deprovision a user (block keys, mark inactive) |
| `/api/v1/admin/users/{user_id}/reprovision` | POST | Re-activate a deprovisioned user |
| `/api/v1/admin/keys` | GET | List all active keys across users (paginated) |
| `/api/v1/admin/keys/{key_id}/revoke` | POST | Admin-revoke any key by ID |
| `/api/v1/admin/audit` | GET | Query audit log with filters and pagination |
| `/api/v1/admin/audit/export` | GET | Export audit log as CSV (max 10,000 rows) |
| `/api/v1/admin/health` | GET | Get scheduler status and background job info |

### Health checks

Two unauthenticated health endpoints are available at the root:

| Endpoint | Description |
|---|---|
| `/health` | Basic liveness check |
| `/ready` | Readiness check (database and LiteLLM connectivity) |

## OpenAPI spec

The backend auto-generates an OpenAPI specification. When running locally:

- **Swagger UI**: `http://localhost:8000/docs`
- **ReDoc**: `http://localhost:8000/redoc`
- **Raw spec**: `http://localhost:8000/openapi.json`
