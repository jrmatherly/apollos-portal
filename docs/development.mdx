<Info>
  **Prerequisites**:
  - Python 3.12+ and [uv](https://docs.astral.sh/uv/)
  - Node.js 24+ and npm
  - Docker and Docker Compose
  - A Microsoft Entra ID app registration (for authentication)
</Info>

## Quick start with Docker

The fastest way to run the full stack locally:

```bash
git clone https://github.com/jrmatherly/apollos-portal.git
cd apollos-portal
mise run dev
```

This starts PostgreSQL, the backend API (with hot reload), the frontend (with HMR), and the docs preview server. The portal is available at `http://localhost:3000`, the API at `http://localhost:8000`, and docs at `http://localhost:3001`.

<Tip>
  `mise run dev` automatically rebuilds Docker images when Dockerfiles change. You can also run `docker compose -f docker-compose.dev.yml up --build` directly.
</Tip>

## Running services individually

### Backend

```bash
cd backend
uv sync
uv run alembic upgrade head   # Run database migrations
uv run uvicorn app.main:app --reload --port 8000
```

The API is available at `http://localhost:8000`. Swagger UI is at `/docs` and ReDoc at `/redoc`.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The frontend dev server starts at `http://localhost:3000` with hot module replacement.

### CLI

```bash
cd cli
uv sync
uv run apollos --help
```

## Environment variables

### Backend

Backend configuration is managed through the `app.config.Settings` class (Pydantic BaseSettings). Create a `.env` file in the `backend/` directory:

```bash
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/apollos
LITELLM_BASE_URL=http://localhost:4000
LITELLM_MASTER_KEY=sk-your-litellm-master-key
AZURE_TENANT_ID=your-tenant-id
AZURE_CLIENT_ID=your-client-id
AZURE_CLIENT_SECRET=your-client-secret
PORTAL_BASE_URL=http://localhost:3000
```

### Frontend

Frontend environment variables use the `VITE_` prefix. Create a `.env` file in the `frontend/` directory:

```bash
VITE_API_BASE_URL=http://localhost:8000
VITE_AZURE_CLIENT_ID=your-client-id
VITE_AZURE_TENANT_ID=your-tenant-id
```

## Running tests

```bash
# Backend tests (from project root)
uv run --package apollos-portal-backend pytest -v

# Backend linting
uv run --package apollos-portal-backend ruff check backend/

# Frontend type checking
cd frontend && npx tsc --noEmit

# Frontend linting
cd frontend && npx biome check .

# Run all checks at once
mise run qa
```

## Database migrations

The backend uses Alembic for database migrations:

```bash
cd backend

# Apply all migrations
uv run alembic upgrade head

# Create a new migration
uv run alembic revision --autogenerate -m "description of change"

# Roll back one migration
uv run alembic downgrade -1
```

## Project structure

```
apollos-portal/
├── backend/           # FastAPI API server
│   ├── app/
│   │   ├── api/v1/endpoints/  # auth, keys, teams, usage, models, settings, provision, admin
│   │   ├── core/              # auth, database, graph client, teams config, logging, exceptions, rate limiting, scheduler
│   │   ├── models/            # SQLAlchemy models (users, keys, teams, audit, notifications)
│   │   ├── services/          # LiteLLM client, provisioning, key lifecycle, email, admin, audit
│   │   └── templates/email/   # Jinja2 email templates (expiry warnings, rotation, deprovisioning)
│   ├── alembic/       # Database migrations
│   └── tests/         # pytest test suite
├── frontend/          # React SPA
│   └── src/
│       ├── components/ # Shared components (Sidebar, ProtectedRoute)
│       ├── contexts/   # React contexts (AuthContext)
│       ├── hooks/      # Custom hooks (useKeys, useUsage, etc.)
│       ├── pages/      # Route pages (Dashboard, ApiKeys, Teams, etc.)
│       └── lib/        # API client, MSAL config
├── cli/               # Click CLI
├── docker/                # Dockerfiles
├── docs/                  # This documentation site (Mintlify)
├── docker-compose.yml     # Production deployment
└── docker-compose.dev.yml # Development (bind mounts, hot reload)
```
