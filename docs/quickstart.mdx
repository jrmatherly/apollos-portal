## Prerequisites

<Info>
  You need a Microsoft Entra ID account with membership in at least one security group that maps to a LiteLLM team. Contact your administrator if you don't have access.
</Info>

## Get started

<Steps>
<Step title="Sign in to the portal">
  Navigate to the Apollos AI portal and click **Sign in with Microsoft**. You'll be redirected to Microsoft's login page for authentication.

  After signing in, the portal automatically provisions your account by mapping your Entra ID security group memberships to LiteLLM teams.
</Step>

<Step title="Create an API key">
  1. Go to **API Keys** in the sidebar
  2. Click **Create key**
  3. Give your key a name (e.g., "development")
  4. Select the team to associate the key with
  5. Copy the key — it won't be shown again

  <Warning>
    Store your API key securely. You cannot retrieve it after creation — only create a new one.
  </Warning>
</Step>

<Step title="Make your first request">
  Use your API key to send a request through the LiteLLM proxy:

  ```bash
  curl -X POST https://your-litellm-proxy/chat/completions \
    -H "Authorization: Bearer sk-your-api-key" \
    -H "Content-Type: application/json" \
    -d '{
      "model": "gpt-4o",
      "messages": [
        {"role": "user", "content": "Hello, world!"}
      ]
    }'
  ```

  The proxy routes your request to the appropriate LLM provider based on your team's model access configuration.
</Step>

<Step title="Monitor usage">
  Go to **Usage** in the sidebar to track your token consumption, request counts, and costs broken down by model and team.
</Step>
</Steps>

## Using the CLI

You can also authenticate and check your profile from the command line:

```bash
# Set up the CLI
cd cli && uv sync

# Configure your tenant and portal URL
uv run apollos configure

# Authenticate with device code flow
uv run apollos login

# Check your portal profile
uv run apollos me
```

See the [CLI overview](/cli/overview) for the full command reference.
