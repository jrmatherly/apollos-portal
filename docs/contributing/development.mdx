This guide covers the full development setup for contributing to the Apollos AI codebase. For a quicker overview, see the [development page](/development).

## Prerequisites

| Tool | Version | Purpose |
|---|---|---|
| Python | 3.12+ | Backend and CLI |
| uv | Latest | Python package management |
| Node.js | 24+ | Frontend |
| Docker | Latest | PostgreSQL and full-stack dev |
| mise | Latest | Task runner (optional) |

## Repository structure

The project uses a uv workspace with three packages:

- `backend/` — `apollos-portal-backend` package
- `cli/` — `apollos-cli` package
- Root `pyproject.toml` — workspace configuration

## Running tests

### Backend

```bash
# Run all tests
cd backend && uv run pytest -v

# Run a specific test file
uv run pytest tests/test_auth.py -v

# Run with coverage
uv run pytest --cov=app -v
```

### Frontend

```bash
# Type checking
cd frontend && npx tsc --noEmit

# Linting
cd frontend && npx biome check .

# Auto-fix lint issues
cd frontend && npx biome check --write .
```

### All checks

```bash
# Run the full QA suite
mise run qa
```

## Database

PostgreSQL 18 runs via Docker Compose. The backend uses SQLAlchemy 2.0 with async support (asyncpg driver) and Alembic for migrations.

```bash
# Start just the database
docker compose up db

# Run migrations
cd backend && uv run alembic upgrade head

# Create a new migration after model changes
cd backend && uv run alembic revision --autogenerate -m "description"
```

## Entra ID app registration

To test authentication locally, you need a Microsoft Entra ID app registration with:

- **Redirect URI**: `http://localhost:5173` (SPA type)
- **API permissions**: `User.Read`, `GroupMember.Read.All`
- **Client secret** for backend service-to-service calls

Set the corresponding environment variables in `backend/.env` and `frontend/.env` as described in the [development guide](/development).
