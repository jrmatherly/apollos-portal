Contributions to Apollos AI are welcome. This guide covers how to submit changes to both the codebase and documentation.

## Prerequisites

| Tool | Version | Purpose |
|---|---|---|
| Python | 3.12+ | Backend and CLI |
| uv | Latest | Python package management |
| Node.js | 24+ | Frontend |
| Docker | Latest | PostgreSQL and full-stack dev |
| mise | Latest | Task runner (optional) |

See the [development guide](/development) for full setup instructions, environment variables, and project structure.

## Code contributions

### Getting started

1. Fork and clone the repository
2. Follow the [development setup](/development) to get your local environment running
3. Create a branch for your changes
4. Make changes and run the test suite
5. Submit a pull request

### Pre-commit checks

Run all checks before submitting a pull request:

```bash
mise run qa
```

This runs backend tests (pytest), frontend tests (vitest), linting (ruff + biome), and type checking (tsc).

### Code style

- **Python**: Ruff handles linting and formatting. Run `uv run --package apollos-portal-backend ruff check --fix backend/` to auto-fix
- **TypeScript**: Biome handles linting and formatting. Run `cd frontend && npx biome check --write .` to auto-fix
- **Commits**: Use conventional commit messages (`feat:`, `fix:`, `chore:`, `docs:`)

### Repository structure

The project uses a uv workspace with three packages:

- `backend/` — `apollos-portal-backend` (FastAPI + SQLAlchemy)
- `cli/` — `apollos-cli` (Click CLI with MSAL device-code auth)
- Root `pyproject.toml` — workspace configuration and shared ruff settings

### Claude Code (optional)

If you use [Claude Code](/ai-tools/claude-code), copy the MCP server configuration:

```bash
cp .mcp.json.example .mcp.json
```

This enables context7 (live documentation lookup), Playwright (browser automation), and GitHub (PR/issue management) MCP servers. The GitHub server requires a Personal Access Token — see the [Claude Code setup guide](/ai-tools/claude-code) for configuration details and available skills.

## Documentation contributions

### Option 1: edit on GitHub

Navigate to the page you want to edit in the `docs/` directory and click the pencil icon to edit directly on GitHub.

### Option 2: local development

1. Install the Mintlify CLI: `npm i -g mint`
2. Navigate to the `docs/` directory
3. Run `mint dev` to preview at `http://localhost:3000`
4. Make your changes — the preview updates automatically
5. Run `mint broken-links` to check for broken links
6. Submit a pull request

### Writing guidelines

- Use active voice and second person ("you")
- Keep sentences concise — one idea per sentence
- Use sentence case for headings
- Bold for UI elements: Click **Settings**
- Code formatting for file names, commands, and code references
- See `docs/AGENTS.md` for full terminology and style conventions

## Entra ID app registration

To test authentication locally, you need a Microsoft Entra ID app registration with:

- **Redirect URI**: `http://localhost:3000/auth/callback` (SPA type)
- **API permissions**: `User.Read` (delegated), `GroupMember.Read.All` (application)
- **Client secret** for backend service-to-service calls

Set the corresponding environment variables in `backend/.env` and `frontend/.env` as described in the [development guide](/development).
