Apollos AI is a self-service portal for managing access to LLM APIs through a [LiteLLM](https://docs.litellm.ai/) proxy. Users authenticate with Microsoft Entra ID, get provisioned into teams based on security group memberships, and manage their own API keys, usage, and settings.

## How it works

<Steps>
<Step title="Authenticate">
  Sign in with your Microsoft Entra ID account. The portal uses PKCE redirect flow for secure browser-based authentication.
</Step>
<Step title="Get provisioned">
  On first login, the portal maps your Entra ID security group memberships to LiteLLM teams. Each team has its own model access, rate limits, and budgets.
</Step>
<Step title="Create API keys">
  Generate API keys scoped to your teams. Use these keys to make LLM requests through the proxy endpoint.
</Step>
<Step title="Monitor usage">
  Track your token consumption, request counts, and costs per model and team from the portal dashboard.
</Step>
</Steps>

## Architecture

The portal consists of three components:

| Component | Technology | Purpose |
|---|---|---|
| **Portal** (frontend) | React 19, Vite 7, TypeScript | Web UI for key management, usage, and settings |
| **API** (backend) | FastAPI, SQLAlchemy, PostgreSQL | REST API, auth validation, LiteLLM proxy integration |
| **CLI** | Click, MSAL, httpx | Command-line interface with device-code auth |

All three components authenticate against Microsoft Entra ID and interact with a LiteLLM proxy instance for LLM routing and budget management.
