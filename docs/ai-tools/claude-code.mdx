Set up Claude Code to work with the Apollos AI codebase, including MCP servers, skills, and documentation tools.

## Prerequisites

- Active Claude subscription (Pro, Max, or API access)
- Node.js 24+ (for MCP servers and frontend tooling)

## Setup

<Steps>
<Step title="Install Claude Code">
  ```bash
  npm install -g @anthropic-ai/claude-code
  ```
</Step>

<Step title="Configure MCP servers">
  Copy the example MCP configuration to enable project-level MCP servers:

  ```bash
  cp .mcp.json.example .mcp.json
  ```

  The example configuration includes three MCP servers. Edit `.mcp.json` to add your GitHub PAT for the GitHub server, or remove servers you don't need.

  <Note>
    `.mcp.json` is gitignored. You can customize your local copy without affecting other contributors.
  </Note>
</Step>

<Step title="Start working">
  ```bash
  claude
  ```

  The project includes built-in skills, subagents, and hooks in the `.claude/` directory. These are available automatically.
</Step>
</Steps>

## MCP servers

The `.mcp.json.example` file configures these servers:

| Server | Purpose | Setup |
|--------|---------|-------|
| [context7](https://github.com/upstash/context7) | Live documentation lookup for project dependencies (FastAPI, SQLAlchemy, React, MSAL, etc.) | Works out of the box |
| [Playwright](https://github.com/anthropics/anthropic-mcp-playwright) | Browser automation for visual testing and E2E verification of frontend flows | Works out of the box |
| [GitHub](https://github.com/anthropics/anthropic-mcp-github) | Structured access to issues, pull requests, and Actions status | Requires `GITHUB_PERSONAL_ACCESS_TOKEN` in `.mcp.json` |

<Tabs>
<Tab title="context7">
  Provides live documentation for any library in the project's dependency tree. Ask questions like "how does FastAPI dependency injection work?" and Claude will fetch current docs instead of relying on training data.

  No configuration required.
</Tab>

<Tab title="Playwright">
  Enables browser automation for testing frontend flows visually. Useful for verifying:
  - The auto-provisioning progress dialog and key display
  - Responsive layout across screen sizes
  - Copy-to-clipboard behavior
  - Animation and transition rendering

  No configuration required — launches a headless browser on demand.
</Tab>

<Tab title="GitHub">
  Provides structured access to the `jrmatherly/apollos-portal` repository without shell commands. Supports:
  - Viewing and creating pull requests
  - Reading and commenting on issues
  - Checking CI/Actions status

  Requires a GitHub Personal Access Token with `repo` scope:

  ```json
  "github": {
    "command": "npx",
    "args": ["-y", "@anthropic/mcp-github"],
    "env": {
      "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_your_token_here"
    }
  }
  ```
</Tab>
</Tabs>

## Skills

| Skill | Command | Purpose |
|-------|---------|---------|
| API doc sync | `/api-doc` | Sync endpoint docs with backend routes |
| New endpoint | `/new-endpoint` | Scaffold a FastAPI endpoint with tests |
| Pre-commit check | `/precheck` | Run full QA suite (tests, lint, types) |
| Migration | `/migration` | Generate and apply Alembic migrations |
| Generate test | `/gen-test` | Scaffold a Vitest + RTL test file for a frontend component |
| Dependency check | `/deps-check` | Audit Python and Node dependencies for updates and vulnerabilities |

## Subagents

| Agent | Purpose |
|-------|---------|
| `security-reviewer` | OWASP-focused security review (auth, CORS, injection, CSV) |
| `docs-reviewer` | Cross-reference docs against code for accuracy |
| `provisioning-tester` | Validate consistency across the provisioning pipeline (backend, frontend, tests, config, docs) |

## Hooks

Hooks run automatically on file edits:

| Hook | Trigger | Behavior |
|------|---------|----------|
| Auto-format | PostToolUse (Edit/Write) | Formats Python (ruff) and TypeScript/MDX (biome) on save |
| Related test runner | PostToolUse (Edit/Write) | Runs co-located `.test.tsx` file when a source `.tsx` file is edited |
| File protection | PreToolUse (Edit/Write) | Blocks edits to `.env`, `teams.yaml`, and lock files |

## Documentation workflow

To contribute to this documentation site using Claude Code:

1. Install the [Mintlify skill](https://mintlify.com/docs/mcp) for component reference and writing standards
2. Read `docs/AGENTS.md` for project-specific terminology and style guide
3. Preview changes with `cd docs && mint dev`
4. Validate before committing with `cd docs && mint validate && mint broken-links`
