Set up Claude Code to work with the Apollos AI codebase, including MCP servers, skills, and documentation tools.

## Prerequisites

- Active Claude subscription (Pro, Max, or API access)
- Node.js 24+ (for MCP servers and frontend tooling)

## Setup

<Steps>
<Step title="Install Claude Code">
  ```bash
  npm install -g @anthropic-ai/claude-code
  ```
</Step>

<Step title="Configure MCP servers">
  Copy the example MCP configuration to enable project-level MCP servers:

  ```bash
  cp .mcp.json.example .mcp.json
  ```

  This configures [context7](https://github.com/upstash/context7) for live documentation lookup of project dependencies (FastAPI, SQLAlchemy, React, MSAL, etc.).

  <Note>
    `.mcp.json` is gitignored. You can add additional MCP servers to your local copy without affecting other contributors.
  </Note>
</Step>

<Step title="Start working">
  ```bash
  claude
  ```

  The project includes built-in skills (`/api-doc`, `/new-endpoint`, `/precheck`, `/migration`) and subagents (`security-reviewer`, `docs-reviewer`) in the `.claude/` directory. These are available automatically.
</Step>
</Steps>

## Available automations

### Skills

| Skill | Command | Purpose |
|-------|---------|---------|
| API doc sync | `/api-doc` | Sync endpoint docs with backend routes |
| New endpoint | `/new-endpoint` | Scaffold a FastAPI endpoint with tests |
| Pre-commit check | `/precheck` | Run full QA suite (tests, lint, types) |
| Migration | `/migration` | Generate and apply Alembic migrations |

### Subagents

| Agent | Purpose |
|-------|---------|
| `security-reviewer` | OWASP-focused security review (auth, CORS, injection, CSV) |
| `docs-reviewer` | Cross-reference docs against code for accuracy |

### Hooks

Hooks run automatically on file edits:

- **PostToolUse**: Auto-formats Python (ruff) and TypeScript/MDX (biome) on save
- **PreToolUse**: Blocks edits to `.env`, `teams.yaml`, and lock files

## Documentation workflow

To contribute to this documentation site using Claude Code:

1. Install the [Mintlify skill](https://mintlify.com/docs/mcp) for component reference and writing standards
2. Read `docs/AGENTS.md` for project-specific terminology and style guide
3. Preview changes with `cd docs && mint dev`
4. Validate before committing with `cd docs && mint validate && mint broken-links`
